# AHI 自适应插值方法：基于理想插值后验误差的 EI 点选择

## 背景

现有 AHI（Adaptive Hybrid Interpolation）方法在 `par_interp_mix.c` 中实现，
通过 `jx_PAMGMeasureAI_ipt` 计算基于 S^T S 图度量的 `AI_measure`，
当 `AI_measure[i] >= mai_threshold` 时将该 F 点标记为 EI 点，
采用长距离插值；否则使用经典修正插值。

## 新方法原理

新增函数 `jx_PAMGMeasureEI_ideal`（`csrc/amg/par_measure_ai.c`），
基于理想插值后验误差思想选择 EI 点。

对每个 F 点 i，计算误差指示子：

```
eta_i = (|a_ii| - sum_{j in C_i} |a_ij|) / |a_ii|
```

其中 C_i 是 i 的强连接 C 点邻居集合（含跨处理器连接 S_offd 部分）。

物理含义：
- eta_i ~ 0：C 点覆盖了行的大部分量，经典插值已足够
- eta_i ~ 1：C 点覆盖不足，需要长距离插值（EI）

与原有方法的区别：

|                | 原有 AHI (ai_ipt=1) | 新方法 (ai_ipt=2) |
|----------------|---------------------|-------------------|
| 度量依据       | S^T S 图论度量     | 理想插值局部残差 |
| 计算代价       | 需 S^T + 直方图排序 | 仅遍历 A 和 S 各行 |
| 物理意义       | 间接（图结构）     | 直接（插值精度） |

## 使用方式

运行时参数 `-ai_ipt 2` 启用新方法，例如：

```
./solver_strong -nts 1 -pid 3 -sid 22 -ai_mt 1 -ai_ipt 2 -ipt 60
```

默认阈值 mai_threshold = 0.3。

## 分析与结论

### Setup 加速效果

- Setup 加速 14.5%（16.83s -> 14.39s）
- 新指示子仅需逐行扫描 A 和 S，比 S^T S 的矩阵转置 + 并行通信 + 直方图排序更轻量
- 加速比：16.83 / 14.39 = 1.17x

### 算子稀疏性

- 新方法在所有层上都选择了更少的 EI 点（全层减少 22.6%）
- 深层（2-4 层）减少尤为显著（35-43%）
- 算子复杂度从 2.557 降至 2.483，RAP 计算和矩阵存储的开销降低

### 收敛与总时间

- 迭代数从 15 增至 18（+20%），但 Setup 加速 14.5% 弥补了部分开销
- 总时间基本持平（42.50s vs 43.44s，+2.2%）
- 对于 Setup 占比更高的大规模问题，新方法的优势会更明显

### 后续优化方向

1. 降低阈值：当前 threshold=0.3，可尝试 0.2 或 0.25，增加 EI 点以改善收敛
2. 层相关阈值：深层矩阵更稠密，可降低阈值以增加 EI 选择
3. 混合策略：粗层用原有 AHI，细层用新方法
4. 并行通信：当前未显式通信跨处理器 CF_marker，仅使用 S_offd 中的 A 值
